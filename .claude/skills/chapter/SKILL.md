# /chapter — генератор глав для «Нежданых гостей»

Этот скилл собирает черновик главы повести в `src/chapters/N. Название.md`.

## Аргументы

- `N` — номер главы (обязательно). Пример: `/chapter 3`.
- Опционально: `--rewrite` — переписать существующую главу с нуля; по умолчанию режим — продолжить/доработать существующий файл, если он есть.
- Опционально: краткая инструкция автора («сделай мрачнее», «добавь сцену в метро», и т.п.).

## Алгоритм

1. **Прочитать контекст**, в таком порядке:
   - `src/план-глав.md` — найти запись для главы N (что должно произойти, какие сцены, какие повороты).
   - `src/сюжет.md` — общий каркас, если плана для N ещё нет.
   - `src/персонажи.md` — кто участвует в этой главе.
   - `src/lore/<нужная цивилизация>/` — если в главе действует кто-то из туристов.
   - `src/dialogs/` — если в главе есть тематический спор/обсуждение, использовать заготовки.
   - Соседние главы в `src/chapters/` — чтобы не нарушить непрерывность и не повторяться.
   - Камертон-фрагмент, если он есть (`src/камертон.md` или подобное) — для тона.
2. **Спросить уточнения**, только если в плане главы есть явные пробелы (нет имени персонажа, неясно место, неясен временной разрыв с предыдущей). Не задавать вопросы, на которые можно ответить из материалов.
3. **Написать главу** соблюдая:
   - **POV**: третье лицо ограниченное от Олега. Никакого «всезнающего» рассказчика, никакого доступа к мыслям других персонажей напрямую.
   - **Формат**: повесть/большой рассказ. Без отступлений и побочных линий, держать темп. Главы скорее короткие, чем длинные (~1.5–4 тыс. слов на главу — ориентир, не жёстко).
   - **Тон**: научная фантастика с юмором, не боящаяся становиться серьёзной. Первая половина книги — больше абсурда и быта; вторая — больше эскалации и моральной философии.
   - **Согласованность с лором**: имена пауков на «Ш», осьминогов на «О», и т.п.; биология и культура цивилизаций — из `src/lore/`.
4. **Сохранить файл** как `src/chapters/N. Название.md`. Название — на русском, без артиклей, лаконичное. Формат имени должен соответствовать `^(\d+)\.\s.+\.md$` (см. `scripts/build-html.mjs`).
5. **Кратко отчитаться** автору: какие сцены вошли, где приняты неочевидные решения, какие пробелы пришлось закрыть на лету (чтобы автор мог зафиксировать или переделать).

## Чего НЕ делать

- Не писать «итоговое примечание автора» в конце главы — только проза.
- Не повторять события предыдущей главы для «контекста читателя».
- Не вводить новых персонажей или мест без записи в плане/материалах — если позарез нужно, сначала спросить.
- Не запускать сборку HTML и не править существующие главы кроме целевой.
- Не использовать emoji в прозе.

## После генерации

Если есть скиллы `/lore-check` или `/voice-check` (могут появиться позже) — упомянуть в отчёте, что их стоит прогнать на свежей главе.
